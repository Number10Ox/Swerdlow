# Swerdlow v0.2 — Design

**Date:** 2026-05-14
**Status:** Draft, awaiting review.
**Authors:** Jon (project owner), Claude (brainstorm partner).
**Pilot input:** [`docs/pilots/2026-05-14-contextdrift-phase-a.md`](../../pilots/2026-05-14-contextdrift-phase-a.md).
**v0.1 design (prior cycle):** [`docs/superpowers/specs/2026-05-14-swerdlow-v0.1-design.md`](2026-05-14-swerdlow-v0.1-design.md).

This is the v0.2 design that addresses pilot finding **F1 (bundle ballooning)** and the closely related **F3 (same target, different work types, different right bundle)**, using **F6 (the user's existing "Load when..." prose tables)** as the design source.

v0.1 + v0.1.1 are shipped. Phase B (wire one consumer path in ContextDrift's SessionStart hook to call `swerdlow context`) is blocked until v0.2 ships a bundle-scoping primitive. v0.2 unblocks Phase B.

---

## 1. Purpose

v0.1's `depends_on:` is a flat untyped edge — every dependency is "always relevant." In a heavily-cross-linked corpus, transitive closure returns ~75% of the indexed set on most non-leaf targets, which is the bundle-ballooning problem ContextDrift's pilot surfaced.

v0.2 introduces **typed edges with free-form mode tags**. Each `depends_on:` entry can declare *when* the edge should be loaded:

```yaml
depends_on:
  - GamePillars                              # always relevant (v0.1 form preserved)
  - {id: MissionGameplay, when: [narration]} # narration sessions only
  - {id: PlanTemplate, when: [plan]}         # plan sessions only
```

Queries become mode-aware:

```bash
$ swerdlow context NarrativeDesign --for narration
```

This filters edges at every traversal hop, cutting branches that don't match the requested mode. The pilot's 23-file `context NarrativeDesign` bundle becomes ~7 files when the corpus is properly annotated.

## 2. User stories

The v0.1 user stories carry forward unchanged (retrofit, maintain over time, get the context bundle, surface in workflow artifacts). v0.2 adds two:

5. **Author scopes edges by work type.** When declaring `depends_on:` on a doc, I can annotate which kinds of sessions need each dependency. The annotation matches the prose convention I already use ("Load when..." columns in design doc tables).

6. **Consumer asks for the bundle relevant to a specific work type.** From a `/narration` session start, I ask `swerdlow context X --for narration` and get the narration-relevant slice of X's deps. From `/plan`, I get the plan-relevant slice. Same doc, different right answer.

**Explicitly NOT in v0.2's stories:**
- Indexing a different graph per mode (mode is a filter, not a separate graph).
- Inferring modes from prose during bootstrap (modes are user-authored).
- Conflict resolution when modes overlap (modes are independent free-form tags, no taxonomy).

## 3. CLI surface

```bash
# v0.1-compatible: no filter, all edges (unchanged)
$ swerdlow context NarrativeDesign

# v0.2 new: filter to single mode (plus always-edges)
$ swerdlow context NarrativeDesign --for narration

# multi-mode: union (narration OR plan OR always)
$ swerdlow context NarrativeDesign --for narration,plan
```

The `--for` flag accepts a comma-separated list of mode tags. No other CLI changes. `init`, `bootstrap`, `bootstrap --apply` are unchanged from v0.1.

## 4. Frontmatter schema

`depends_on:` accepts a **heterogeneous list** where each entry is either a string or a dict. Both forms can coexist in the same list.

**String form (v0.1, preserved):**

```yaml
depends_on:
  - GamePillars
```

Interpreted as `{id: "GamePillars", when: []}` — always relevant.

**Dict form (v0.2 new):**

```yaml
depends_on:
  - {id: MissionGameplay, when: [narration]}
  - {id: SceneAuthoring, when: [narration, plan]}
  - {id: TDD, when: plan}        # convenience: single string coerced to [plan]
  - {id: GamePillars}            # no `when` = always (same as bare string)
```

**Schema rules:**

- `id` is required, must be a string.
- `when` is optional. May be a list of strings, or a single string (coerced to a one-element list), or absent (treated as `[]`).
- Empty `when: []` ≡ absent `when` ≡ bare-string entry. All three mean "always relevant."
- Anything else (entry is a number / bool / dict without `id` / dict with malformed `when`) records an `Issue(type="parse_error")` and the entry is dropped.

**Mode vocabulary:** free-form. Swerdlow ships no known-modes enum. Each project defines its own (e.g., `narration`, `plan`, `design`, `test`). No validation against a known-modes list.

**Case sensitivity:** modes are case-sensitive strings (`narration` ≠ `Narration`). Documented; not auto-normalized.

## 5. Data model

New `Edge` dataclass replaces `tuple[str, str]`:

```python
@dataclass(frozen=True)
class Edge:
    from_id: str
    to_id: str
    when: tuple[str, ...] = ()   # empty tuple = "always"; otherwise mode tags
```

`tuple` (not `list`) so the dataclass stays frozen and hashable.

`Graph` updated:

```python
@dataclass(frozen=True)
class Graph:
    nodes: dict[str, Node]
    edges: list[Edge]              # was: list[tuple[str, str]]
    issues: list[Issue]
```

`Node`, `Issue`, `Proposal`, `BootstrapIssue`, `BootstrapPlan` are unchanged.

**`Issue.type` is unchanged.** Existing kinds (`missing_ref`, `duplicate_id`, `parse_error`, `cycle_detected`) carry over. Malformed dict entries fold into `parse_error`. No new issue kinds in v0.2.

## 6. Loader changes

`_build_edges` is rewritten to normalize each `depends_on` entry:

- String entry → `Edge(from_id, to_id=string, when=())`.
- Dict entry with valid `id` and optional `when` → `Edge(from_id, to_id=dict['id'], when=tuple(when_list))`.
- Malformed entry → `Issue(type="parse_error", doc_id=<from>, detail=<reason>)`.

Pass 1 (node build) is unchanged. Determinism guarantees (lexical path sort, first-wins on duplicate ID) carry forward.

## 7. Context query

`context(graph, doc_id, modes=None)` gains an optional `modes` parameter.

**Filter rule:**

An edge `Edge(from_id, to_id, when=W)` is **allowed** during traversal if:
- `modes is None` (no filter — v0.1 behavior), OR
- `W` is empty (always-relevant), OR
- `W` intersects `modes` (any tag matches).

Otherwise the edge is **cut** for this traversal — as if it didn't exist.

**Algorithm:**

1. Find target node by id. Not found → `ContextError` (CLI exits 1).
2. DFS from target. At each node, recurse into edges that pass the filter rule above (and only those).
3. Visit-set + ancestor-set tracking unchanged from v0.1. Cycles still produce `Issue(type="cycle_detected")` but only for edges that survived the mode filter — a cycle reachable only through filtered-out edges is invisible (desirable).
4. Post-order emit produces the topologically-sorted path list.

**Same-depth ordering** is preserved by walking the `depends_on:` list in declaration order, filtering as we go.

**CLI flag:**

```python
p_ctx.add_argument("--for", dest="for_modes", default=None,
                   help="Comma-separated mode tags to filter edges by")

# Parsed:
modes = args.for_modes.split(",") if args.for_modes else None
```

Modes are passed to `context(graph, doc_id, modes=modes)`. `None` preserves v0.1 behavior.

## 8. Apply + scan + plan file

**Apply:** the ruamel round-trip already handles arbitrary YAML structures byte-preservingly, so user-authored typed entries survive unchanged through apply cycles. New behavior: `add_depends_on` from the plan file can contain dicts (if the user hand-edited the plan to add `when:`). Each entry is written as-is — string stays string, dict stays dict. Idempotency continues to subtract by ID, ignoring mode tags.

**Scan:** unchanged user-facing behavior. Bootstrap still proposes bare-string entries from prose markdown links. Users add mode annotations during the plan-edit step (or after apply via direct edit of the frontmatter).

**Plan file:** `Proposal.add_depends_on: list[str | dict[str, Any]]` — heterogeneous list. PyYAML round-trips both forms transparently. No new plan-file schema.

## 9. CLI edge cases

**Unknown mode:** `--for nonexistent` does not error. Returns the always-only subgraph (which includes the target and any always-edges transitively). Stderr emits a one-time warning per unknown mode:

```
warning: mode 'nonexistent' is not present on any edge in the corpus
```

The warning is computed once at query time by checking all edges' `when` tuples for membership.

**Empty bundle from filter:** if a filter leaves the target with no edges and the target has no body content, just the target's path prints. Not an error.

**Mixed bare/typed entries in same frontmatter:** fully supported. The loader handles each entry independently.

## 10. Module impact

| Module | Change |
|---|---|
| `swerdlow/types.py` | Add `Edge` dataclass; change `Graph.edges` type to `list[Edge]` |
| `swerdlow/loader.py` | `_build_edges` rewritten to parse string/dict entries and produce `Edge` instances |
| `swerdlow/context.py` | `context()` gains `modes` parameter; filter rule applied during DFS |
| `swerdlow/cli.py` | `--for` flag on `context` subcommand; passes parsed modes to `context()` |
| `swerdlow/bootstrap/scan.py` | `_existing_dep_ids` helper that handles both string and dict entries (for idempotency) |
| `swerdlow/bootstrap/apply.py` | Idempotency check extracts IDs from both forms; existing typed entries preserved via ruamel round-trip |
| `swerdlow/bootstrap/plan.py` | `Proposal.add_depends_on` type widened to allow dicts; PyYAML handles round-trip |
| `swerdlow/config.py` | No change |

## 11. Testing approach

`pytest`, fixture-driven, additive on top of v0.1's 65 tests.

**Loader tests:**
- Bare-string entries produce `Edge(when=())` — backward compatibility.
- Dict entry `{id: foo, when: [narration]}` produces `Edge(when=("narration",))`.
- Single-string `when: narration` coerces to `("narration",)`.
- Missing `id` in dict → `parse_error` issue, entry dropped.
- Non-list, non-string `when` (e.g., `when: 42`) → `parse_error`.
- Mixed bare/typed in same `depends_on` works.

**Context tests:**
- No filter (`modes=None`) returns same bundle as v0.1.
- Single-mode filter returns target + always-edges + matching-mode edges.
- Multi-mode filter returns union of all matching modes.
- Edge cut at hop N stops traversal beyond it (the "filtering at every hop" rule).
- Unknown mode returns always-only bundle.
- Mode filter interacts correctly with cycles (cycle only via filtered-out edge → no cycle issue).
- Same-depth declaration order preserved through filter.

**Scan tests:**
- Idempotency works against typed entries in existing frontmatter (re-scan doesn't re-propose).

**Apply tests:**
- Typed entries in source frontmatter survive a round-trip apply (byte-preserving).
- Plan with dict entry in `add_depends_on` writes a dict into source frontmatter.
- Plan with mixed string/dict entries writes both forms correctly.

**E2E test:**
- Extension of the v0.1 e2e test: same workflow, but the apply step writes some typed entries and `context X --for <mode>` returns the expected scoped bundle.

## 12. Distribution

No changes to install, packaging, dependencies, or Python version requirement. v0.2 = v0.1.1 + the changes above.

## 13. Settled decisions (from brainstorm)

- **Primary mechanism: typed edges (per-edge metadata), not doc-level scope tags.** Mirrors the user's existing prose-table convention (F6).
- **Schema: `{id, when}` only — no `role` / `note` fields.** YAGNI; no consumer needs them yet.
- **`when` semantics: OR over modes.** `when: [narration, plan]` matches `--for narration` and `--for plan`.
- **CLI flag: `--for narration,plan` comma-separated.** One flag, multi-value.
- **Modes are free-form strings, project-defined.** No enum, no validation.
- **Filtering applies at every traversal hop, not just the root.** F1 is solved by aggressive cutting.
- **v0.1 schema fully backward-compatible.** Bare-string entries are valid, always-relevant. No forced migration.
- **Data model: `Edge` dataclass.** Replaces tuples. Type-safe and frozen.
- **No new `Issue` kinds.** Malformed typed entries fold into `parse_error`.
- **Scope: typed edges + mode scoping only.** Visualize / reverse / check / MCP defer to v0.3.

## 14. Out of scope for v0.2

Deferred to v0.3+ (no order implied):

- Graph rendering (`swerdlow graph` → Graphviz / Mermaid output).
- Reverse lookup (`swerdlow reverse <code-path>` → which docs govern this file).
- `swerdlow check` (cycle detection, orphan docs, dead-ref report — useful but distinct from query-time issues).
- MCP server.
- LLM-priming surface (skill / plugin that primes agents on Swerdlow usage in a project).
- Indirection mechanisms for F2 (Now.md deps rot weekly). Mode scoping helps partially but doesn't fully solve "active deliverable changes weekly." Revisit after Phase B.
- Index-vs-dep heuristic (B3 from pilot). Possible add via "links in tables / under specific headings = candidate suppressions."
- Bootstrap heuristics beyond markdown links (wiki-syntax, prose patterns, bare filenames).
- Per-section deps. Big architectural change; defer.
